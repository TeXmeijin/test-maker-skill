# マークダウン記法の詳細仕様

Test Maker MCP に渡すマークダウン（`create_examination` / `update_examination` の `markdown` 引数）の完全仕様。

> **最新の公式仕様はサーバー側の `mcp__test-maker__get_test_maker_format_spec` ツールで取得できます**。このファイルはオフラインで参照するためのスナップショットなので、差異がある場合はツール側の応答が正。

---

## 1. サポートするマークダウン記法

- GFM（GitHub Flavored Markdown）準拠
- 見出し（`#`, `##`, `###`）
- 順序付きリスト（`1.`）/ 順序なしリスト（`-`）
- テーブル（`|` 区切り）
- 強調（`**bold**`, `*italic*`, `` `code` ``）
- リンク（`[text](url)`）
- 数式（`$...$`, `$$...$$`, `\(...\)`, `\[...\]` — KaTeX でレンダリング）
- `<question>` タグ（穴埋め・選択・多答問題）

記述式問題（`<write-question>`）は現時点で MCP 非対応。必要な場合は編集ページで追加。

## 2. `<question>` タグの基本ルール

### 必ず文の一部として埋め込む

Editor の制約上、`<question>` タグだけの行はレンダリングされない。必ず文脈のある文章の中に埋め込む。

- NG: `<question correct="apple" />`（単独行）
- OK: `りんごの英語は<question correct="apple" />です。`

### 数式デリミタの内側に入れない

`$...$`, `$$...$$`, `\(...\)`, `\[...\]` のいずれの内側にも `<question>` を入れない。KaTeX が数式として解釈して文字列として描画されてしまう。

## 3. `<question>` の全属性

| 属性 | 必須 | 説明 |
|---|---|---|
| `correct` | yes | 正解。文字列 or JSON 配列文字列 |
| `choiceItems` |  | 選択肢。JSON 配列文字列（シングルクォートで囲む） |
| `isMultipleChoice` |  | 複数選択式なら `"true"`、単一選択なら `"false"` |
| `correctItems` |  | 多答問題の正解配列（`isMultipleChoice="true"` とセット） |
| `hasAlternativeSolutions` |  | 別解ありモード。`"true"` / `"false"` |
| `explain` |  | 解説文 |
| `hint` |  | ヒント |
| `ruby` |  | ルビ（漢字の読みなど） |
| `score` |  | 配点 |

属性値のクォート規則:

- 通常の値はダブルクォートで囲む: `correct="東京"`
- JSON 配列を値に取る属性（`correct`, `choiceItems`, `correctItems`）は、**配列全体をシングルクォートで囲み、内側の各要素はダブルクォート** を使う: `choiceItems='["banana", "orange", "grape"]'`
- これは HTML 属性値のエスケープ回避のため。外側ダブルクォート + 内側ダブルクォートにすると属性が壊れる

## 4. 出題形式ごとの例

### 穴埋め

```markdown
赤血球に含まれる酸素運搬タンパク質は<question correct="ヘモグロビン" />である。
```

### 単一選択

```markdown
光合成が行われる細胞小器官は<question correct="葉緑体" choiceItems='["ミトコンドリア", "葉緑体", "リボソーム", "ゴルジ体"]' isMultipleChoice="false" />である。
```

### 正誤判定

```markdown
白血球は核をもたない。<question correct="誤り" choiceItems='["正しい", "誤り"]' isMultipleChoice="false" />
```

### 複数選択（多答）

```markdown
交感神経が優位なときの反応として正しいものをすべて選べ。<question correctItems='["心拍数の増加", "気管支の拡張"]' choiceItems='["心拍数の増加", "消化管運動の促進", "気管支の拡張", "瞳孔の縮小"]' isMultipleChoice="true" />
```

### 別解あり（正解が複数の表記で成立する場合）

```markdown
日本の与党第一党は<question correct='["自民党", "自由民主党"]' hasAlternativeSolutions="true" />である。
```

### ヒント・解説つき

```markdown
日本の首都は<question correct="東京" hint="日本最大の都市" explain="東京は関東地方に位置する日本の首都" />である。
```

### 数式とセット

```markdown
円の面積を求める公式は $S = \pi r^2$ である。半径が 3 のとき、面積は<question correct="9π" />。
```

## 5. 問題作成時の品質ルール

### 5.1 問題文から答えを推測可能にしない

NG: 問題文に答えの性質が書かれている（例: 「大幅に減少」と書かれた文の穴埋めに ▲10% が選択肢にあると自明）。

OK: 選択肢側に数値の説明を含めるなどして推測を封じる。

### 5.2 誤答選択肢の作り方

- **数字**: キリの良い数字ばかりにせず、同時期の他の出来事の年号など根拠のある選択肢を混ぜる
- **文字**: 正解と「似せた選択肢」と「普通の選択肢」をバランス良く混ぜる
- 無関係な選択肢ばかりだと「消去法で正解がわかる」ので避ける

### 5.3 完全一致で採点できる問題のみ

穴埋め・選択・多答はシステムが **完全一致** で自動採点するため、答えが一意に定まる問題のみ作成する。「あなたの考え」等の開放的な問題は記述式扱いになるため MCP 非対応（編集ページから追加）。

### 5.4 別解ありの使いどころ

- 同義語が存在する用語（アボガドロ数 / アボガドロ定数）
- 表記揺れがある語句（原子量 / 分子量 / 式量のいずれも正解の場合）
- 送り仮名の揺れ（行なう / 行う）
- カナ表記の揺れ（ニーハオ / ニイハオ）

## 6. タイトル・問題数の推奨値

- タイトル: **5 文字以上 300 文字以内**
- 公開テストの問題数: **5 問以上**推奨（Test Maker の検索インデックス対象になる条件）
- 1 テストの問題数上限: 無料プラン 30 問 / PRO 上限なし

## 7. 完成例

```markdown
## 日本史 — 江戸時代の基礎テスト

1603年、<question correct="徳川家康" />が征夷大将軍に任命され、江戸幕府を開いた。

鎖国体制のもと、日本と貿易関係にあったオランダとの窓口となった港は<question correct="出島" choiceItems='["長崎", "出島", "博多", "堺"]' isMultipleChoice="false" />である。

徳川家康の死後、江戸幕府第2代将軍となったのは<question correct='["徳川秀忠", "秀忠"]' hasAlternativeSolutions="true" />である。
```
