# マークダウン記法の詳細仕様

Test Maker MCP に渡すマークダウン（`create_examination` / `update_examination` の `markdown` 引数）の完全仕様。

> **最新の公式仕様はサーバー側の `mcp__test-maker__get_test_maker_format_spec` ツールで取得できます**。このファイルはオフラインで参照するためのスナップショットなので、差異がある場合はツール側の応答が正。

---

## 1. サポートするマークダウン記法

- GFM（GitHub Flavored Markdown）準拠
- 見出し（`#`, `##`, `###`）
- 順序付きリスト（`1.`）/ 順序なしリスト（`-`）
- テーブル（`|` 区切り）
- 強調（`**bold**`, `*italic*`, `` `code` ``）
- リンク（`[text](url)`）
- 数式（`$...$`, `$$...$$`, `\(...\)`, `\[...\]` — KaTeX でレンダリング）
- `<question>` タグ（穴埋め・選択・多答問題）

記述式問題（`<write-question>`）は現時点で MCP 非対応。必要な場合は編集ページで追加。

## 2. `<question>` タグの基本ルール

### 必ず文の一部として埋め込む

Editor の制約上、`<question>` タグだけの行はレンダリングされない。必ず文脈のある文章の中に埋め込む。

- NG: `<question correct="apple" />`（単独行）
- OK: `りんごの英語は<question correct="apple" />です。`

### 数式デリミタの内側に入れない

`$...$`, `$$...$$`, `\(...\)`, `\[...\]` のいずれの内側にも `<question>` を入れない。KaTeX が数式として解釈して文字列として描画されてしまう。

## 3. `<question>` の全属性

| 属性 | 必須 | 説明 |
|---|---|---|
| `correct` | yes | 正解。文字列 or JSON 配列文字列 |
| `choiceItems` |  | 選択肢。JSON 配列文字列（シングルクォートで囲む） |
| `isMultipleChoice` |  | 複数選択式なら `"true"`、単一選択なら `"false"` |
| `correctItems` |  | 多答問題の正解配列（`isMultipleChoice="true"` とセット） |
| `hasAlternativeSolutions` |  | 別解ありモード。`"true"` / `"false"` |
| `explain` |  | 解説文 |
| `hint` |  | ヒント |
| `ruby` |  | ルビ（漢字の読みなど） |
| `score` |  | 配点 |

属性値のクォート規則:

- 通常の値はダブルクォートで囲む: `correct="東京"`
- JSON 配列を値に取る属性（`correct`, `choiceItems`, `correctItems`）は、**配列全体をシングルクォートで囲み、内側の各要素はダブルクォート** を使う: `choiceItems='["banana", "orange", "grape"]'`
- これは HTML 属性値のエスケープ回避のため。外側ダブルクォート + 内側ダブルクォートにすると属性が壊れる

## 4. 出題形式ごとの例

### 穴埋め

```markdown
赤血球に含まれる酸素運搬タンパク質は<question correct="ヘモグロビン" />である。
```

### 単一選択

```markdown
光合成が行われる細胞小器官は<question correct="葉緑体" choiceItems='["ミトコンドリア", "葉緑体", "リボソーム", "ゴルジ体"]' isMultipleChoice="false" />である。
```

### 正誤判定

```markdown
白血球は核をもたない。<question correct="誤り" choiceItems='["正しい", "誤り"]' isMultipleChoice="false" />
```

### 複数選択（多答）

```markdown
交感神経が優位なときの反応として正しいものをすべて選べ。<question correctItems='["心拍数の増加", "気管支の拡張"]' choiceItems='["心拍数の増加", "消化管運動の促進", "気管支の拡張", "瞳孔の縮小"]' isMultipleChoice="true" />
```

### 別解あり（正解が複数の表記で成立する場合）

```markdown
日本の与党第一党は<question correct='["自民党", "自由民主党"]' hasAlternativeSolutions="true" />である。
```

### ヒント・解説つき

```markdown
日本の首都は<question correct="東京" hint="日本最大の都市" explain="東京は関東地方に位置する日本の首都" />である。
```

### 数式とセット

```markdown
円の面積を求める公式は $S = \pi r^2$ である。半径が 3 のとき、面積は<question correct="9π" />。
```

## 5. 問題作成時の品質ルール

### 5.1 問題文から答えを推測可能にしない

穴埋めの大原則は **「問題文だけ読んで答えに辿り着けないこと」**。次のパターンが頻発するので明示的に潰す。

#### パターン A: 答えの性質が問題文に書かれている

問題文に答えのヒント（増加/減少・大小・正負など）を書いてしまうと、選択肢の中で条件に合うものが一つしかなく自明になる。

- ❌ 悪い例（「大幅に減少」が答えのヒントになっている）

  ```markdown
  2025年の貿易収支は前年度と比べ<question correct="▲10%" choiceItems='["▲10%", "12%", "▲0.2%"]' isMultipleChoice="false" />と大幅に減少した。
  ```

- ✅ 良い例（数値と説明文をセットで選択肢にする）

  ```markdown
  2025年の貿易収支は前年度と比べ<question correct="▲10%と大幅に減少" choiceItems='["▲10%と大幅に減少", "12%と大幅に増加", "▲0.2%と堅調に推移"]' isMultipleChoice="false" />した。
  ```

**原則**: 問題文に数値の性質（増加/減少、大きい/小さいなど）を書く場合は、選択肢にもそれぞれの説明を付ける。

#### パターン B: 答えの語が同じ題材内のどこかに露出している

タイトル・リード文・先行する別の問題文・引用元の見出しなど、**同じテスト内で答えの語そのものが既出**になっていないか必ず確認する。題材（記事・教科書・PDF）の文言をそのまま流用すると起きやすい。

- ❌ 悪い例（タイトルに答えの語「指示追従性」が既出）

  ```markdown
  ## 「指示追従性」が上がったAIコーディングの世界で

  筆者が注目している性能特性は<question correct="指示追従性" />である。
  ```

- ✅ 良い例（題材の文言を機械的に転記せず、答えの語が前後に出ない問題文に書き直す）

  ```markdown
  ## 最近のAIコーディング動向

  筆者は「言われた指示にどれだけ正確に従えるか」を示す<question correct="指示追従性" />という性能特性に注目している。
  ```

**原則**: 答えの語は **同じテスト内で他の場所に文字列として出現しない** ようにする。題材原文に出ている語を穴埋めの答えにする場合は、周辺の文言を言い換える。

### 5.2 誤答選択肢の作り方

選択式問題は **「正解だけ明らかに浮く」** と消去法で解けてしまい、テストとして機能しなくなる。逆に **全部が正解に近すぎる** と意地悪な引っ掛けになり、知識を測れない。次のバランスを取る。

#### 数字の選択肢

キリの良い数字ばかり並べると正解だけが浮く。逆に正解の前後の年号で固めると区別不能になる。**同時代の実在の出来事の年号など、根拠のある数字を混ぜる** のが定石。

- ❌ 悪い例（キリが良すぎて正解だけ異質）

  ```markdown
  夏目漱石『こころ』が発表されたのは<question correct="1914" choiceItems='["1905", "1910", "1920"]' isMultipleChoice="false" />年である。
  ```

- ❌ 悪い例（前後で固めすぎて知識では判別不能）

  ```markdown
  夏目漱石『こころ』が発表されたのは<question correct="1914" choiceItems='["1912", "1915", "1917"]' isMultipleChoice="false" />年である。
  ```

- ✅ 良い例（同時代の実際の出来事の年号で誤答を構成）

  ```markdown
  夏目漱石『こころ』が発表されたのは<question correct="1914" choiceItems='["1912", "1917", "1923"]' isMultipleChoice="false" />年である。
  ```

  - 1912: 明治天皇崩御 / 1917: ロシア革命 / 1923: 関東大震災 など、すべて実在の事件に対応する年号。

#### 文字の選択肢

カテゴリの偏りに注意する。**1つだけ毛色が違う選択肢があると正解だけ目立つ**し、**全部が同じカテゴリだと不自然に絞られて見える**。「普通の選択肢」と「正解に似せた選択肢」を混ぜる。

- ❌ 悪い例（K だけ異質で正解バレバレ）

  ```markdown
  「先生」が墓地で手を合わせていたのは、<question correct="K" choiceItems='["奥さん", "父親", "母親"]' isMultipleChoice="false" />の墓である。
  ```

- ❌ 悪い例（全部1文字の人物名で不自然）

  ```markdown
  「先生」が墓地で手を合わせていたのは、<question correct="K" choiceItems='["K", "J", "M"]' isMultipleChoice="false" />の墓である。
  ```

- ✅ 良い例（普通の答え + 似せた答えのバランス）

  ```markdown
  「先生」が墓地で手を合わせていたのは、<question correct="K" choiceItems='["奥さん", "K", "J"]' isMultipleChoice="false" />の墓である。
  ```

#### 一般原則

- 誤答は「根拠のある選択肢」と「正解に似せた選択肢」を適切に混ぜる
- 明らかに異質な選択肢（カテゴリ違い・桁違い）は入れない
- 全部が正解に酷似していて常識的な知識では区別できないものも避ける
- 無関係な選択肢ばかりだと消去法で正解が割れるので避ける

### 5.3 完全一致で採点できる問題のみ

穴埋め・選択・多答はシステムが **完全一致** で自動採点するため、答えが一意に定まる問題のみ作成する。「あなたの考え」等の開放的な問題は記述式扱いになるため MCP 非対応（編集ページから追加）。

### 5.4 別解ありの使いどころ

- 同義語が存在する用語（アボガドロ数 / アボガドロ定数）
- 表記揺れがある語句（原子量 / 分子量 / 式量のいずれも正解の場合）
- 送り仮名の揺れ（行なう / 行う）
- カナ表記の揺れ（ニーハオ / ニイハオ）

## 6. タイトル・問題数の推奨値

- タイトル: **5 文字以上 300 文字以内**
- 公開テストの問題数: **5 問以上**推奨（Test Maker の検索インデックス対象になる条件）
- 1 テストの問題数上限: 無料プラン 30 問 / PRO 上限なし

## 7. 完成例

```markdown
## 日本史 — 江戸時代の基礎テスト

1603年、<question correct="徳川家康" />が征夷大将軍に任命され、江戸幕府を開いた。

鎖国体制のもと、日本と貿易関係にあったオランダとの窓口となった港は<question correct="出島" choiceItems='["長崎", "出島", "博多", "堺"]' isMultipleChoice="false" />である。

徳川家康の死後、江戸幕府第2代将軍となったのは<question correct='["徳川秀忠", "秀忠"]' hasAlternativeSolutions="true" />である。
```
