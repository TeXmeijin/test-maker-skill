# トラブルシューティング

Test Maker MCP でよくある問題と対処法。

---

## 認証・接続

### `401 Unauthorized` / トークン期限切れ

**症状**: ツール呼び出しで「token expired」「再認可が必要」のメッセージ。

**対処**:
- Claude Code: `/mcp` → test-maker を選んで再認可
- Cursor / Claude Desktop: MCP サーバー設定画面から再接続
- それでもダメなら AI クライアントを再起動して、再度ツールを呼び出す（初回呼び出しで OAuth フローが走る）

### 「MCP server disconnected」と出る

**対処**:
- クライアントを再起動
- URL が正しいか確認: `https://api.test-maker.app/mcp`（末尾スラッシュなし、`http` ではなく `https`）
- 企業ネットワーク等で blocking されていないか確認

## 枠制限（無料プラン）

### `非公開テストの上限（3件）に到達しています`

**対処（ユーザーに提示すべき選択肢）**:
1. 既存の非公開テストを `update_examination` で上書きする
2. 公開テストとして作成し直す（公開枠にまだ空きがあれば、同じ内容を `isPublic: true` + `tagId` で再作成）
3. PRO にアップグレード: https://www.test-maker.app/m/payment
4. 既存テストを削除したい場合は **編集ページから削除**（MCP には削除機能が無い）

### `公開テストの上限（10件）に到達しています`

同上。非公開枠が空いていれば非公開で作成できる。MCP 経由では削除できないので、Web UI で整理する案内を。

## `tagId` エラー

### `tagId="..." が存在しません`

**対処**:
- `list_examination_tags` を呼んで利用可能な id 一覧を取得
- それでも合うカテゴリが無ければ、非公開テスト（`isPublic` を省略 / false）にすれば `tagId` は不要

### 公開テストなのに `tagId` が必須と言われる

- `isPublic: true` で作成・更新する場合 `tagId` は必須
- タグ候補: `english`（英語）/ `math`（数学）/ `science`（理科）/ `japanese`（国語）など
- 最新は `list_examination_tags` で確認

## マークダウンの解析エラー

### `問題が 1 問も抽出できませんでした`

**原因**: `<question>` タグが本文中に 1 つも無い、またはパースに失敗している。

**対処**:
- タグを文中に埋め込んでいるか確認（単独行だとパースされない）
- 属性値のクォートが崩れていないか確認（特に `choiceItems` は `'[...]'` のシングルクォート）
- それでもダメなら `get_test_maker_format_spec` を呼んで最新仕様を確認

### `<question>` タグが「文字列」としてそのまま表示される

**原因**: 数式デリミタ（`$`, `$$`, `\(`, `\[`）の内側にタグを入れている。

**対処**: 数式とタグを別に分ける。

```markdown
# NG
$S = \pi r^2$ において半径が<question correct="3" />のとき。

# OK（数式とタグを分離）
$S = \pi r^2$ と表される式で、半径が<question correct="3" />のときを考える。
```

## Round-trip（get_examination → 編集 → update_examination）

### 取得したマークダウンと元のマークダウンで、属性の順番や配列内のスペースが違う

**これは正常**。意味的に等価なら問題なく保存される。

具体的には:
- 入力: `<question correct="東京" hint="..." explain="..." />`
- 取得: `<question correct="東京" explain="..." hint="..." />`

- 入力: `choiceItems='["A", "B", "C"]'`
- 取得: `choiceItems='["A","B","C"]'`

見た目は変わるが完全一致採点には影響しない。

### 特定ノードが round-trip で消える

Plate JSON → Markdown への逆変換は主要ノード（見出し・段落・リスト・`<question>`・数式・hr）を網羅するが、一部（表・画像・デザインカスタマイズ・記述式問題など）は round-trip で再現されない場合がある。

**対処**:
- round-trip が不完全なケースは MCP 向けではない。編集ページで直接調整する
- 記述式問題を使いたい場合は編集ページで追加（MCP 非対応）

---

## 他人のテストを操作したい

MCP 経由で操作できるのは **認証ユーザー自身が作成したテストのみ**。

- `get_examination` / `update_examination` で他人の examinationId を指定するとエラー
- 共同編集機能は MCP 非対応（編集ページから共同編集者を追加する必要あり）

---

## 記述式問題（手動採点）を作りたい

現時点で MCP 非対応。将来の対応は検討中。

**回避策**: 編集ページ（`https://www.test-maker.app/m/create-exam/{id}`）から記述式問題を追加できる。MCP でベースのテストを作成したあと、編集ページで記述式問題を足すフローを案内する。

---

## デザインカスタマイズ・配信期間・ログイン必須にしたい

MCP 非対応（将来検討中）。編集ページから設定可能:

- デザイン → 配色・背景・フォント変更
- 配信期間 → 開始日・終了日
- 回答者ログイン必須 → パスワード保護も同時に可能
- 採点結果の表示制御 → 本番試験直前など

詳しくは `get_feature_catalog` ツールを呼ぶか、編集ページを開いて設定。
